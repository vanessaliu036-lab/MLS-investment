## 1. 系統目標(Objective)

本系統**禁止僅依據「資金流入」或「資金流出」直接判斷市場多空。**

AI 必須分析五個維度:

| 維度 | 對應 Level |
|---|---|
| 資金品質(Money Quality) | Level 2 |
| 市場結構(Market Structure) | Level 3 |
| 籌碼承接(Absorption) | Level 4 |
| 賣壓來源(Selling Pressure Source) | Level 5 |
| 隔日延續機率(Continuation Probability) | Level 7 |

所有分析必須建立在**多維度交叉驗證**(Level 8.1)。

## 4. Level 1 — Capital Flow Analysis(資金流分析)

### 4.1 Sector Flow(族群資金)

每個族群必填欄位:

| 欄位 | 單位 | 計算方式 |
|---|---|---|
| 今日資金流 | 億元 | 當日族群內成分股淨流入加總 |
| 近 5 日累積 | 億元 | 滾動 5 日加總 |
| 近 10 日累積 | 億元 | 滾動 10 日加總 |
| 近 20 日累積 | 億元 | 滾動 20 日加總 |
| 排名變化 | 名次 | 對比昨日的族群排名升降 |
| 是否持續流入 | True/False | 連續 5 日淨流入為正才算 True |

趨勢星級:
- 5 日累積 > 0 且排名上升 ≥ 3 名:★★★★★
- 5 日累積 > 0 且排名不變:★★★★
- 5 日累積 > 0 但排名下降:★★★
- 5 日累積 ≤ 0:★★

**範例**:
```
IC設計 | Today +239億 | 5 Days +618億 | 10 Days +1035億 | 排名↑5 | Trend ★★★★★
```

### 4.2 Stock Flow(個股資金)

每檔股票必填:

| 欄位 | 單位 |
|---|---|
| 成交金額 | 億元 |
| 成交量 | 張 |
| 估算主動買 | 億元 |
| 估算主動賣 | 億元 |
| 估算淨流入 | 億元 |
| **Flow Score** | 0~100 |

## 11. Level 8 + 8.1(決策防火牆)

### 11.1 Level 8 基本校驗

分析完成後 AI 必須自行檢查:
> 「**是否因為資金流入就直接判斷看多?**」

如果是 → 重新分析。

### 11.2 Level 8.1 Evidence Triangulation(強化驗證)

#### 五類證據支柱

| 代號 | 類別 | 範例指標 |
|---|---|---|
| **A** | 資金面 | 估算淨流入 / 法人買賣超 / 融資增減 |
| **B** | 價量結構 | 收盤價位置 / 成交量極端值 / 委託單失衡 |
| **C** | 技術面(趨勢) | 站穩 MA5/MA10 / 創高創低 / KD/MACD 背離 |
| **D** | 市場結構(空間) | 族群相對強弱 / 大盤貢獻度 / 產業位階 |
| **E** | 外部事件 | 國際盤連動 / 產業消息 / 總經數據 |

**強制**:AI 必須從五類中**至少選取三個「異質」類別**作決策依據。

#### 偽三源 vs 真三源

```
❌ 偽三源(全部來自 A):
「淨流入 50 億 + 法人買超 40 億 + 融資增加」
→ 視為單一指標,結論無效,強制退回重做。

✅ 真三源(三個異質類別):
「淨流入 50 億(A) + 站回 MA5 且收高(C) + 族群為大盤最強前 3 名(D)」
→ 通過門檻,可產出結論。
```

### 11.3 衝突處理協議(Consensus Protocol)

比對三項證據的方向:

| 證據比例 | 處置 |
|---|---|
| **方向一致(3:0 或 2:1 且主流明確)** | 給出「偏多 / 偏空」明確建議,標示強度(強 / 中 / 弱) |
| **方向矛盾(1:1:1 或 2:1 但主流強度極弱)** | **不得給明確買賣建議**,**強制轉為「等待確認(Pending Confirmation)」**,必須寫出: |

**「等待確認」強制輸出格式**:

```
多方證據:______(舉 1 項);
空方證據:______(舉 1 項);
中立證據:______(舉 1 項)。
三方矛盾,今日無明確方向。

下一驗證訊號為:
明日開盤 30 分鐘內是否站穩今日收盤價 ±0.5% 區間。
```

#### 「主流強度極弱」量化(TBD-6)

> **請 Vanessa 拍板**:
> **候選 A**:三項證據中,主流方的領先項目分數差距 < 5 分 = 極弱
> **候選 B**:主流方 2 項證據的綜合影響分 < 60 = 極弱
> **候選 C**:你自己給具體值

## 12. AI Decision & Evidence Log(原 Final Output 第 11 項)

### 12.1 強制輸出欄位

```
採用證據:
  ① 資金流(正向)— 淨流入 50 億,站穩 5 日均量
  ② 量價結構(背離)— 量增價跌,疑似換手
  ③ 法人(中立)— 外資小幅買超 5 億、投信賣超 8 億

衝突指數:
  2 正向 vs 1 中立(無反向)→ 主流方向:正向
  主流強度(計算):60.5

最終決策:
  ✅ 明確偏多(強度:中)
  或
  ✅ 明確偏空(強度:弱)
  或
  🔍 等待確認(Pending Confirmation)
    觸發重新進場的「價格條件」:今日收盤價 +2% 以上
    觸發重新進場的「時間條件」:明日 09:30 前 30 分鐘內
```

### 12.2 與 `/rankings` 對接的字串格式(EOD Parse 用)

為了讓 EOD 管線可以正則抓取,Final Output 第 11 項**「最終決策」必須包含以下固定關鍵字**:

| 標籤類型 | 字串格式 | 榜單顯示 |
|---|---|---|
| 偏多 | **最終決策:明確偏多(強度:強/中/弱)** | 隔日 ✓ 漲 |
| 偏空 | **最終決策:明確偏空(強度:強/中/弱)** | 隔日 ✗ 跌 |
| 等待確認 | **最終決策:等待確認(Pending Confirmation)** | 隔日 🔍 訊號確認中 |

**EOD parse 規則**:

```python
# 偽代碼,正式實作在 eod_pipeline.py
match = re.search(r"最終決策:(明確偏多|明確偏空|等待確認)", report_text)
if match:
    decision = match.group(1)
    # 寫入 DB 的 ai_decision 欄位
```
