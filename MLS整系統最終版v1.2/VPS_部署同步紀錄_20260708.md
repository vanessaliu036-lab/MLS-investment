### 改動 2：起 mls-legacy container
**build image**：`sudo docker compose build mls-legacy`（28 秒，shioaji 1.5.5 + uvicorn 0.50.2 進 image）
**起服務**：`sudo docker compose down && sudo docker compose up -d`

### 改動 3：server.py — 加 lifespan 啟動 scheduler
**檔案**：`/opt/pos-v1.2/server.py`
**問題**：Dockerfile CMD 是 `uvicorn server:app`（module mode），`if __name__ == "__main__"` **永遠不會跑**。結果 scheduler_loop thread 從未被啟動 → `/api/state` 永遠 `{"status":"starting"}`、板塊資料全空。
**修法**：加 FastAPI `@asynccontextmanager lifespan`，在 lifespan 內 `db.init()` + `threading.Thread(target=scheduler_loop, daemon=True).start()`。

```python
from contextlib import asynccontextmanager

@asynccontextmanager
async def lifespan(app):
    db.init()
    threading.Thread(target=scheduler_loop, daemon=True).start()
    yield

app = FastAPI(title="MLS Standard", lifespan=lifespan)
```

### 改動 4：server.py — 路由重排（`/`=主系統）
**問題**：原本 `/` = sectors.html (簡化版)、「主系統」首頁被指到 `/legacy`。
**修法**：
- `@app.get("/")` → index.html (34833B 主系統)
- `@app.get("/sectors")` → sectors.html (新增的板塊個股頁)
- `@app.get("/legacy")` → sectors.html (後備)
